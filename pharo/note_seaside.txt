Chemin d'une requête sur Seaside.


ZnZincServerAdaptor startOn: 8081

Lance un ZnManagingMultiThreadedServer qui écoute sur le port 8081
Quand il reçoit une requête, dans le message ZnManagingMultiThreadedServer >> executeOneRequestResponseOn:, 
la requête est transformé de ZdcSocketStream en ZnRequest par (request := self readRequestSafely: stream timing: timing)
puis prise en charge par handleRequest:

Ensuite il appelle self authenticateAndDelegateRequest: request dans la méthode ZnManagingMultiThreadedServer >> handleRequestProtected: 
Dans cette methode, il appelle (delegate handleRequest: request), delegage étant un ZnSeasideServerAdaptorDelegate

On a ensuite ZnSeasideServerAdaptorDelegate >> handleRequest: znRequest
	"I pass a Zinc HTTP Components ZnRequest to my Seaside Server Adaptor using #process:"
	^ self adaptor process: znRequest

adaptor étant un ZnZincServerAdaptor

ZnZincServerAdaptor >> process: aNativeRequest transforme un ZnRequest en WARequestContext
dans la lignecontext := self contextFor: aNativeRequest. puis ensuite appelle self handle: context;

ZnZincServerAdaptor >> handleRequest: aRequestContext et le defaultRequestHandler est WADispatcher default
